# Device I/O

Device I/O connects the portable identity engine to its physical or digital eyes
and canvas.

This branch will contain:

- digital peer-canvas readers used by the first web prototype;
- physical camera discovery, capture, and stable routing adapters;
- canvas/display output, fullscreen recovery, and heartbeat control;
- calibration loading and device-capability reporting;
- hot-plug, reconnect, unavailable-device, and fallback behavior;
- simulated devices for development without exhibition hardware.

Hardware serial numbers, credentials, and venue-specific routes belong in secure
deployment configuration rather than identity packages.
