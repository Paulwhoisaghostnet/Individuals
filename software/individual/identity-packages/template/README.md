# Template identity package

This package demonstrates the minimum authored contents of a future Individual.
It is deliberately unformed: its neutral placeholder perception and drawing rules
must be replaced before it becomes an exhibited identity.

To create an Individual:

1. Copy this directory using a lowercase, stable identity ID.
2. Rewrite `manifest.ts`, especially origin, private narrative, traits, ideal self,
   ideal physical form, initial embodied self-perception, perception limits,
   drawing limits, and statement.
3. Add cognition instructions under `prompts/` without including credentials or
   live private memory.
4. Add owned visual material and its provenance under `assets/`.
5. Define identity-specific perception and drawing policies.
   The perception policy must have a unique model ID and bounded tuning controls;
   defaults should express the intended exhibited character.
6. Add tests proving recognizable continuity and intentional constraints.
7. Assemble the package with production or simulated adapters in the runtime.

An identity package is not complete merely because its manifest typechecks. It
must communicate an authored point of view and establish limits that remain
visible across cycles.
