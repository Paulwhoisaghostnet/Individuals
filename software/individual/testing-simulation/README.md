# Testing and simulation

Simulation allows the complete identity loop to be developed and validated before
physical devices or paid model services are available.

This branch will contain:

- deterministic cognition, perception, drawing, and memory adapters;
- simulated peer societies and portrait-routing fixtures;
- virtual cameras, canvases, network conditions, and clocks;
- restart, corruption, latency, missing-peer, and power-loss scenarios;
- golden artifact and provenance checks;
- long-duration cycle and memory-growth tests.

Production adapters must pass the same behavioral contracts as their simulations.
