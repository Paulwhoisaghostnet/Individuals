# Hardware operations requirements

An installation is complete only when it can be commissioned, observed, repaired,
and handed over. Operational requirements should be designed alongside the active
hardware rather than added after installation.

Initial requirements:

- **OPS-001:** Every installation must have a repeatable commissioning and
  acceptance procedure.
- **OPS-002:** Hardware health must be observable without exposing controls in the
  public exhibition interface.
- **OPS-003:** Failure alerts must identify the affected Individual and component.
- **OPS-004:** Routine replacement must preserve identity data and calibration.
- **OPS-005:** Critical spares and restoration time targets must be defined before
  public opening.
- **OPS-006:** Changes to approved hardware must be recorded and revalidated.

See `commissioning/`, `monitoring/`, and `spares/`.
